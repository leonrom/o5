Состояния объектов

Подвисабельные объекты могут иметь следующие признаки состояния (задаются в aO5.act):

  • uScroll — тег будет обрабатываться при скроллинге,-. т.е. он может наезжать на зафксированные (если у него самого неопределён xFixed). Изменяется IntersectionObserver'ом в модуле DoInit для нефиксированных объектов.
  • canFix — уменшились размеры видимости тега в напрвлении подвисания и он стал кандидат на фиксацию на границах контейнеров.
  • wasFull — тег был полностью видимым на экране,- без этого ему не установится canFix и он не будет фиксироваться .
  • xFixed — фиксированность объекта, принимающая значенияё:
    • null - не фиксирован
    • true - фиксирован при 'наезде' на другой зафиксированный объект.
    • tag - фиксирован при 'вылете' за границы контейнера <tag>.
    Пересчитывается при каждом сколлинге (если включен контроль скроллирования) для всех объектов с установленным uScroll.
  • oldIR — предыдущая видимость тега (т.е. значение intersectionRatio)

Модуль DoInit

Модуль контролирует поведеник подвисабельных тегов на эране:

  • создаёт обозреватель obsrv0 с обработчиком Obsrv0();
  • подключает к обозревателю obsrv0 все теги, содержащие класс 'olga5_shp' при отсутствии (у этого класса) квалификатора ':N' (т.е. 'none')

Обработчик DoInit.Obsrv0() выполняет:

    если увеличилась видимость тега (т.е. intersectionRatio > oldIR)
    • Если aO5shp не определён (т.е. это первое увеличение с нулевого значения)
      • инициирует тег: создаёт в нём объект aO5shp;
      • добавляет aO5shp в массив wshp.aO5s.
    • устанавливается для тега (точнее, для его aO5shp.act) признак uScroll=true.
    • если тег стал полностью видимым (т.е. intersectionRatio==1)
      • сбрасывается признак canFix (т.е. задаётся canFix=false)
      • устанавливаеся признак wasFull (задаётся wasFull=true)
      • если среди всех (зарегистрированых в wshp.aO5s) тегов отсутствуют с canFix, то отключается контроль скроллинга.
    если уменшилась видимость тега (т.е. intersectionRatio < oldIR
    • если wasFull==true и уменшение видимости произошло 'в направлении' подвисания cls.dirV :
      • устанавливается canFix
      • если список контейнеров пуст (т.е. aO5.parents.length == 0) - формируем цепочку контейнеров, а также определяем списки frame's и owner's, запустив функцию PO5shp()
      и, если контроль скроллинга не включен, то
      • включается контроль скроллинга
      • запускается DoScroll() отработки скроллинга
      • иначе
    • если тег стал полностью НЕвидимым (т.е. intersectionRatio==0) сбрасываются признаки: uScroll= false и wasFull= false
    запоминает oldIR =intersectionRatio.

Здесь включить (отключить) контроль скроллинга означает window.addEventListener('scroll', DoScroll, true) ( window.removeEventListener('scroll', DoScroll, true))
