Подключение всплывающих окон.

Скрипт o5pop.js используется для показа поясняющих примеров во всплывающих окнах в случаях, когда открытие нового окна/вкладки представляется избыточным.

В отличие от большинства скриптов библиотеки Olga5  данный скрипт может использоваться автономно,- как без подключения ядра o5com!.js так и при его подключении, но при этом скрипт д.б. размещен выше скрипта ядра.

1. Подключение всплываемого окна

Модуль позволяет повесить всплытие окна на одно или несколько произвольных событий тега. Если событие не является onClick, то (вероятно) придётся в самом браузере выдавать специальное разрешение на появление всплывающих окон (т.к. браузеры считают безопасными только те окна, которые пользователь вызвал самолично,- осознанным нажатием клавиши либо кнопки мыши,- в частности click'ом по некоторому тегу. Подробнее см. ниже п.7).

Всплытия окон обеспечиваются функцией PopUp, сохраняемой в объекте window.olga5. Например, чтобы по клику на слове показать (т.е. по событию onClick) открывалось окно, содержащее файл add/popup.html, его можно записать:

<span onClick="window.olga5.PopUp(this, './add/popup.html', 'width: 211, height: 211')> показать </span>.

Закрытие всплывшего окна выполняется click'ом на вызвавшем теге, независимо от события, вызвавшего его открытие.

2. Визуализация состояния всплытия окна

Поскольку модуль позволяет держать всплытыми несколько окон, в нём предусмотрено визуальное отображение ссылок на теги, вызывающие эти всплытия.

По умолчанию модуль добавляет в документ CSS стили отображения тегов, могущих вызвать всплытие окон.
Так, для тегов, обрабатывающих событие onClick, добавляется правило: 

.olga5_popup, .olga5_popup_Act, 
.olga5_popup_PopUp{cursor: pointer;}

Для тегов, содержащих класс olga5_popup (либо атрибут o5popup) добавляются правила:

  • пассивного состояния тега 
    .olga5_popup    {cursor: pointer;color: black;background-color: lavender;
           border-radius: 4px;border: 1px dashed gray;}
span.olga5_popup {padding-left: 4px;padding-right: 3px;}
img.olga5_popup  {border: none;background-color: transparent;position: relative;}
.olga5_popup_errArg {opacity:0.5;}
  • при задании ненулевого значения атрибута o5timer (см. п.5) добавляется мигание тега при всплытом окне 
    .olga5_popup.olga5_popup_act {outline-offset: 2x;animation: blink XXXms infinite linear;}
@keyframes blink {
    99% {outline: 2px dashed  black;outline-offset: 2x;}66% {outline: 3px dashed  white;}
    33% {outline: 2px dashed  black;}0% {outline: 3px dashed white;outline-offset: -2x;}
}

Здесь 'XXX' - подставляемое автоматически значение o5timer, пересчитанное в миллисекунды.

Добавление (удаление) класса olga5_popup_act для тега выполняется при всплытии (закрытии) дополнительного окна,- независимо от наличия у тега класса olga5_popup.

После появления всплытого окна оно становится активным, т.е. будет обрабатывать все клавиатурные команды. Поэтому управление основным окном при этом возможно только кликом мышкой.

3. Аргументы вызова всплываемого окна

Функция PopUp получает три строковых аргумента:

  1. ссылка на сигнальный тег, например 'this', на котором отображаеся (миганием) наличие всплывающего окна;

    Если ссылка на сигнальный тег отсутствует или есть пустой строкой, то таковым предполагается текущий (содержащий вызов PopUp()) тег.

  2. адрес загружаемого файла

    Если значение адреса файла не пустое и не содержит один из символов ['\', '/', '.', '#', ], то оно считается внутренней ссылкой на текущий документ и автоматически дополняется его URL-адресом.

    Пустое значение адреса файла не считается ошибкой,- т.к. если тегу задан id то список параметров такого тега может использоваться как контейнер с умалчиваемыми значениями для других тегов.
    При подключении модуля совместно с o5com!.js (ядром библотеки) этот адрес файла может содержать именованные ссылки.

    Адрес может содержать параметры после символов '?' (для задания атрибутов скрипта) и/или '#' (для позиционирования на документе). Еесли указаны оба, то первым д.б. '?'.

  3. список дополнительных параметров.

    Cписок содержит пары имя:значение (вместо ':' можно '='), разделённые символом ','. При отсутствии у пар значения (т.е. разделяющего символа) их имена считаются id-именами ссылочных тегов, чьи параметры вызова PopUp() будут взяты дополнительно.

    Пары разделяются символом ',' или ';' (они тут эквивалентны), а имена и значения - любым '=' либо ':'. Вместо left, top, width, height и group можно писать сокращенно их первые буквы (l, t, h и g без учета регистра).

3.1. Аргументы вызова всплываемого окна

Пары имя:значение (при наличии ':') в списке дополнительных параметров определяют список параметров открываемого окна. Такой список может (не) содержать любые стандартные атрибуты спецификации window.open(), например width:720 и height:800.

В программе прописаны следующие умалчиваемые параметры согласно спецификации, описывающие:

  • геометрию окна,- left=-22, top=11, width=555, height=333
  • альтернатива предыдущим,- screenx: null, screeny: null, innerwidth: null, innerheight: null
  • поведение окна,- alwaysRaised: 1, alwaysOnTop: 1, menubar: 0, toolbar: 0, status: 0, resizable: 1, scrollbars: 0
Эти значения м.б. изменены скриптовым атрибутом o5params (см.п.5).

В параметрах width и height значения могут быть как безымянными (т.е. подразумеваемыми в пикселах), так и иметь указанной единицу измерения '%' - относительно размеров активного экрана. Также единица '%' может указываться для параметров left, top,- в этом случае подразумевается положение центра всплываемого окна (а не его края), а позиционирование окна обеспечивается таким образом, чтобы оно было целиком видно на экране активного экрана.

При нескольких дисплеях под 'активным' экраном подразумевается тот дисплей, на котором в основном расположено окно браузера. При позиционировании окна его координаты устанавливаются с учетом доступной зоны экрана, при отрицательных значениях left и top они берутся относительно правой или нижней сторон этой зоны.

При первоначальном всплытии могут игнорироваться заданные размеры окна, например, в FireFox всплывшее окно по умолчанию показывается в максимизированном видк.

3.2. Дополнительные параметры всплытия окон

Дополнительно к стандартным могут задаваться параметры:

  • group - строка с именем группы из которой на экране может быть только одно всплывшее окно. Нулевое, пустое или отсутствующее значение интерпретируется одинаково как '0'.
  • text - строка, содержащая 'текст тега' (т.е. значение тегового атрибута innerHTML) при всплытом окне. При отсутствии параметра или его значения 'текст тега' не изменяется. Не должен содержать ',' или ';';
  • head - альтернативный заголовок всплываемого окна, появляющийся при его (заголовка) мигании. При отсутствии значения - мигания заголовка не будет.

Эти параметры используются скриптом для синхронизации отображения всплывших окон

Если ссылочный тег содержит несколько вызовов PopUp() или атрибутов popup, то параметры будут браться у первого из них. Поэтому в ссылаемых тегах желательно обрабатывать не более одного события.

Значением по-умолчанию каждого дополнительного параметра есть пустая строка. При его задании значение не должно содержать символы ',' или ';', а кавычки в начале и в конце - игнорируются. Регистр игнорируется. Рекомендуется эти параметры, (а также nocss,- см. ниже п.4) употреблять с присвоением значения, либо избегать использования их названий в документе в качестве id тегов.

3.3. Имена ссылочных тегов

Параметры, заданные в списке дополнительных параметров без разделителя ':' (или '=' ) являются именами ссылочных тегов, в которых скрипт будет брать недостающие (незаданные) параметры.

При формировании строки параметров всплываемого окна скрит сначала ищет недостающие поочерёдно в ссылочнх тегах, а в конце берёт из умалчиваемых параметров (см. п. 3.1). Умалчиваемые значения прошиты в скриптте, но могут быть изменеты при вызове скрипта (см. п. 5. Подключение скрипта).

Примеры использования ссылочных тегов можно посмотреть, кликнув: автономно или в составе библиотеки.

4. Использование атрибута o5popup

Для упрощенния обработки события onClick вместо указания обработчика события можно задавать атрибут o5popup, являющийся обёрткой вызова (описанной выше) функции PopUp() для события click с автоматическим добавлением класса olga5_popup (определяющий визуальное отображение тега). Значением является строка аргументов, разделённых символами ';' и/или ','.

Первым аргументом является url открываемого окна (аналог адреса файла для PopUp). Последующие аргументы повторяют список дополнительных параметров функции PopUp

Аналог сигнального тега, предусмотренного для PopUp() отсутствует среди аргументов, т.к. при o5popup таковым всегда подразумевается текущий тег.

В дополнение к списку параметров открываемого окна (из п. 3) может задаваться nocss (или коротко,- 'n'), запрещающий добавление класса olga5_popup (как и при наличии тегового атрибута o5nocss,- см. ниже п.5)

Вместо вызова

<span class="olga5_popup" onClick=
"window.olga5.PopUp(this, 'тесты+./add/popup.html', 'width: 211, height: 211')"> показать </span>,

записываем:

<span o5popup="тесты+./add/popup.html; width: 211, height: 211")> показать </span>,

Тут жирным шрифтом выделены необходимые атрибуты.

При незаданном url'е (в т.ч. при отсутствии разделителя  ';') значение o5popup используется только для подстановки (по id тега) в другие вызовы всплывающих окон аналогично как PopUp().

Допускается использование атрибута o5popup в теге <a>. В этом случае адрес в атрибуте, как правило, игнорируется и тег отрабатывает стандартным образом. Лишь при отсутствии в этом теге атрибута href будет выполняться обработка атрибута o5popup.

5. Подключение скрипта

Подключение скрипта o5pop.js выполняется в заголовке (в секции <head>) HTML (или PHP) файла:

<script src="../olga5/o5pop.js" o5params="width:666px" o5nocss=0 o5timer=0</script>

Скрипт может содержать необязательные атрибуты:

  • o5params - значения умалчиваемых параметров всплываемых окон;
  • o5nocss - указывается с ненулевым значением если требуется не подключать CSS-правила, встроенные в модуль o5pop;
  • o5timer - интервал мигания тегов, вызвавших всплытие и возможно заголовка всплывшего окна в секундах. При наличии параметра его умалчиваемое значение 0.7с. Значения меньше 0.1с округляются до 0 - при котором миганий не будет. Атрибут имеет смысл только при ненулевом значении o5nocss.

Подключение скрипта в составе библиотеки olga5 (т.е. если среди подключаемых скриптов имеется o5com!.js) даёт следующие преимущества:

  • в обрабатываемых тегах html-файла могут использоваться символические ссылки;
  • в системном журнале при задании o5debug протоколирование выполняется более красочно.

Оба варианта подключения приведены в следующем пункте.

6. Создание "Списка Литературы"

Использование атрибута o5popup позволяет создавать подобие "списка литературы". Для этого на странице должен находиться список тега <ol> с атрибутом 'id="o5contents"'. Отдельные элементы <li> этого списка могут содержать теги с атрибутом o5popup либо href. Те элементы, на которые имеются ссылки в тексте обязательно должны иметь уникальный id для ссылки на него, что позволяет менять содержимое списка не затрагивая остальную часть документа.

В самом же документе можно в атрибутах o5popup ссылаться на эти теги по их id. Для этого вместо ссылочного адреса (первого элемента списка в o5popup) следует писать "id= XXX", где XXX - значение id у элемента "списка литературы".

В значение тега, который ссылается на "список литературы" будет добавлен номер литературы из списка, обрамлённый квадратными скобками, а само название будет высвечиваться во всплывающей подсказке.

Параметры всплываемого окна формируются конкатенацией параметров атрибутов o5popup вызывающего тега и элемента списка литературы. Это позволяет однократно определить параметры окна для всех ссылок на элемент списка

Теги в "списке литературы" могут не содержать атрибут o5popup либо содержать его с пустым адресом. В этих случаях ссылочный адрес ищется в атрибуте href тега списка.

Ознакомиться с рабочим (черновым) примером можно по ссылке pop1.html.

7. Демонстрационные примеры

С автономной работой модуля можно ознакомиться на примере файла pop1.html, где скрипт o5pop.js функционирует без использования ядра библиотеки o5com!.js

Ниже приведены примеры работы модуля o5pop> с использованием ядра библиотеки o5com!.js.

Напоминание: при появлении всплывших окон они захватывают фокус на себя, поэтому нажатия клавиш действуют на активное всплывшее окно, а управление основным окном осуществляется только мышкой.

1) Примеры вызова всплытия пустого окна (в автономном режиме в окне отображается файл './add/popup.html'):

  • - клик по тегу <input> с заданным атрибутом o5popup и параметрами окна
    top =1, left: 9999; width=222, h:111; group="".
    Здесь, для примера, смешано использование символов: '='|':' и ';'|',', а также вместо 'height' указано 'h'
    Данный тег имеет id="p1" и на его параметры будут ссылки в последующих примерах;
  • 'открыть' - клик по локальному тегу <span> с классом olga5_popup с заданным onClick и параметрами
    PopUp(this, 'тесты+add/popup.html', 'group=0, t:111, p1, text:закр. окно');
  • 'открыть' - клик по тегу <span> с классом olga5_popup и заданным onClick и параметрами
    PopUp(this, 'https://rombase.ru/o5/olga5-tests/add/popup.html', 'group=0, t:177, p1, text:закр. окно')

2) Варианты вызова PopUp() при различном задании первого параметра (задан class="olga5_popup"):

  1. ...PopUp('p01', 'тесты+add/popup.html', 'p1')   - мигает часть заголовка данного абзаца,- тег с id='p01';
  2. ...PopUp(this, 'тесты+add/popup.html', 'p1')    - мигает вызывающий тег;
  3. ...PopUp(p01, 'тесты+add/popup.html', 'p1')     -   как 1;
  4. ...PopUp('', 'тесты+add/popup.html', 'p1')       -  как 2;
  5. ...PopUp(   'тесты+add/popup.html', 'p1')       -   показывает 'не ту' ссылку (пропущен ссылочный тег);
  6.    o5popup='тесты+add/popup.html; p1'           -    аналог 2;
  7. ...PopShow(444, 333, 'тесты+add/popup.html')    -    аналог 2, но меньшего размера.

В последнем примере показно использование еще одной обёртки над функцией PopUp.

В примерах параметр group либо отсутствует, либо задан с нулевым или пустым значением. Эти варианты интерпретируются одинаково как group='0', поэтому всплытие очередного окна закрывает предыдущее всплывшее окно. Для закрытия такого всплывшего окна следует кликнуть в любом месте вызывающего окна.

3) Примеры по наведению курсора (надуманные) всплытия с разных доменов,- конкретноый [адрес] приводится во всплывающей подсказке (после наведения курсора) по двум различным событиям на одном <img>-теге , где -

  • наведение курсора (см. ниже Пояснение) обрабатывается событием onMouseEnter с аргументами (всплытие внизу в разных местах):
    this, '[адрес]', ' p2, left:[0, 50 или 99]%, top:99%, group:p22[a, b или c]'
    Закрытие окна - повторно навести курсор на иконку.
  • клик обрабатывается атрибутом (всплытие вверху в разных местах):
    "o5popup="[адрес]; p2, group:p21[a, b или c]""
    Закрытие окна - повторный клик по иконке.
Здесь 'p2' - имя некоторого тега <p> с атрибутом o5popup="width:33.3%,height:33.3%,left:33.3%,top:0%", из которых left и top переопределяются в самих тегах.

Пояснение к всплытию по наведению курсора: из соображений безопасности оно по-умолчанию отключено в браузерах. Его можно разрешить, но в разных браузерах - по-разному. Например, в Chrome нужно кликать по окну каждый раз перед наведением курсора, а в FireFox достаточно подтвердить разрешение для данного сайта. В приведенных примерах при возникновении такой блокировки иконка становится бледной, а во всплывающем (по наведению курсора на иконку) подсказке сообщается об этом.

4) Еще примеры размещения тегов в разных углах экрана:

  • - левый верхний угол, мигающий заголовок окна;
  • - правый верхний угол, заголовок окна не мигает;
  • - правый нижний угол, текст тега меняется на 'закрыть'.

Здесь параметры вызова всплывающих окона написаны прямо на 'кнопках'. Наличие кавычек в значениях параметров - игнорируется.

5) Демонстрация вызовов всплывших окон при кликах на вложенных тегах:

"./add/popup.html;p1,top:10,left:33%,group:1"
окно - вверху, левее
"./add/popup.html;p1,top:-10,left:67%,group:2"
окно - внизу, правее
"./add/popup.html;p1,t:-50%,l:50%,g:2"
окно в центре, сокращены названия параметров
Напоминание: фокусировка всплывших окон может не срабатывать, поэтому следует отключить полноэкранный режим основного окна.

8. Дополнительные пояснения

Использование всплывающих окон считается небезопасным для пользователя. Некогда бывало, что попав (разумеется,- случайно) на какой-то порносайт, браузер мгновенно переполнялся всплывшими окнами настолько, что приходилось перезагружать систему.

Ныне браузеры позволяют всплытие лишь по событию click, т.е. по т.с. 'осознанному' действию пользователя. Для обеспечения всплытий по иным событиям пользователю необходимо дать специальное разрешение в настройках браузера.

Способы позволения всплытия окон по наведению курсора для браузеров описаны в интернете, но часто они уже устаревшие. Поэтому, при использовании такого всплытия, следует пояснять пользователям,- как в справке самого браузера ввести текст "Всплывающие окна" (или его варианты ☺) и действовать по (например, такой) инструкции.

Так, например, сообщение в console.log об "ошибка создания всплывающенго окна: возможно дан 'http' а не 'httpS'..." в браузере Mozilla было устранено добавлением сайта в список "Allowed to send pop-ups and use redirects" разрешенных в настройках Settings -> Privacy and security.

Дополнительной проблемой всплывших окон является необходимость их постоянной (автоматически программой) фокусировки при любой активации основного окна. Да и не всегда фокусировка срабатывает, что особенно неприятно, когда основное окно занимает весь экран. Данный скрипт попытался решить данную проблему, но не гарантирует постоянное отображение всплывших окон во всех браузерах.

Пожалуй, использование для выдачи дополнительной информации именно всплывающих окон не есть хорошая идея. Оно имеет смысл, когда окно браузера занимает лишь часть экрана (экранов), а всплывшее окно помещается на свободное место. Да и то, часть пространства будет теряеться на обязательные заголовок окна и адресную строку, требуемые браузерами по соображениям безопасности.

В данной версии библиотеки использование всплывающие осталось как попытка реализации древнего windows-подобного интерфейса. Ну и как расширение тега <a>. В дальнейшем они, вероятно, будут заменены более гибкими плавающими блоками,- подобными реализованному в модуле o5dbg.
