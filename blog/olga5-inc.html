Вставка html-файлов или их фрагментов в текст загруженного документа

Скрипт нескольлко расширяет возможности тега <iframe> при вставке html-файлов или их фрагментов, находящихся в одном домене с основным документом.

Вставка выполняется добавлением содержимого считанного файла к содержимому тега, где считывание реализовано с использованием 'веб API интерфейса' XMLHttpRequest. В документ могут одноуровнево вставляться несколько файлов. При наличии в считываемом файле тегов <body> загружается содержимое первого из них (иначе - весь файл). Вставляться может как весь загруженный текст, так и лишь содержимое отдельного селектора.

Чтобы в тег вставить содержимое (или его часть) некоторого файла этот тег должен содержать атрибут o5include, значением которого является строка с именем файла после которого может быть символ '?' с последующим селектором тега в этом файле.

Посмотреть реализацию подгрузки разных фрагментов из двух разных файлов можно здесь, где для атрибута o5include показаны следующие вариатны значений:

  • './inc1.html?.cls2' — класс cls2, независимо от наличия квалификаторов;
  • './inc1.html?.cls2:aa' — класс cls2 и квалификатором aa (т.е. значение атрибута class должно содержать указанную подстроку);
  • './inc1.html?#id1' — с id='id1';
  • './inc1.html?span' — тег span;
  • './inc1.html?[moe]' — атрибут moe,- в квадратных скобках м.б. любое допустимое выражение;
  • './inc1.html?' — включение файла при пустом селекторе;
  • './inc2.html' — включение файла при отсуцтствии селектора

    • Вставляется содержимое первого тега (загруженного файла), удовлетворяющего заданному выражению. В примере были показаны ставки в теги div, однако можно вставлять в любые теги (дополняя при этом их содержимое).

    Разумеется, сами файлы inc1.html и inc2.html считываются только по одному разу.

    На загрузку файлов с "чужиих доменов" действуют ограничния 'CORS policy'.

    При подключении скрипта совместно с иными библиотечными скриптами он должен располагаться прежде них. Также модуль может выполняться автономно,- без подключения ядра библиотеки o5common.js,- как это сделано в упомянутом примере. Результаты загрузки файлов приводятся в сообщении "Обработка 'CInclude'" в console.log.
