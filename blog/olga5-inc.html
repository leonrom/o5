Вставка html-файлов или их фрагментов в текст загруженного документа

Скрипт частично расширяет возможности тега <iframe> при вставке html-файлов или их фрагментов, находящихся в одном домене с основным документом.

Вставка выполняется добавлением содержимого считанного файла к содержимому тега, где считывание реализовано с использованием 'веб API интерфейса' XMLHttpRequest. В документ могут одноуровнево вставляться несколько файлов. При наличии в считываемом файле тегов <body> загружается содержимое первого из них (иначе - весь файл). Вставляться может не только весь загруженный текст, но и лишь содержимое отдельного селектора.

Здесь (кликните) можно посмотреть пример организации нескольких вставок из двух файлов.

Чтобы в тег вставить содержимое (или его часть) некоторого файла этот тег должен содержать атрибут o5include, значением которого является строка с именем файла после которого может быть символ '?' с последующим селектором тега в этом файле. Например: 

		<div o5include='./inc1.html'     ></div>
		<div o5include='./inc1.html?#i3' ></div>
		<div o5include='./inc1.html?.s1' ></div>
		<div o5include='./inc1.html?span'></div>

Здесь показаны четыре вставки. В первой вставляется всё содержамое тега <body> из файла inc1.html, а в остальных - только первых соответствющих тегов: с id=i3, class=s1 и наименованием тега span. Конечно, при этом сам файл inc1.html считывается только один раз.

В примере показана вставка в теги div, однако можно вставлять в любые теги (заменяя при этом их содержимое).

На загрузку файлов с иных адресов действуют ограничния 'CORS policy'.

При модключении скрипта совместно с иными библиотечными скриптами он должен располагаться прежде них. Также модуль может выполняться автономно,- без подключения ядра библиотеки o5common.js,- как это сделано в упомянутом примере. Результаты загрузки файлов приводятся в сообщении "Обработка 'CInclude'" в console.log.
