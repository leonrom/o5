Вставка html-файлов или их фрагментов в текст загруженного документа

Скрипт несколько расширяет возможности тега <iframe> при вставке html-файлов или их фрагментов, находящихся в одном домене с основным документом.

Вставка выполняется добавлением содержимого (или его части) считанного файла к содержимому тега, содержащего атрибут 'o5include'. Вставляться может как весь загруженный текст (ntu body), так и лишь содержимое некоторого тега, определяемого селектором.

Считывание файла реализовано с использованием 'веб API интерфейса' XMLHttpRequest. На загрузку файлов с "чужиих доменов" действуют ограничния 'CORS policy'.

Чтобы вставить только часть считываемого файла (т.е. содержимое его тега <body>) следует после атрибута 'o5include' написать '?' и селектор начинающийся символом:

  • [ - выбирается тег, удовлетворяющий querySelectorAll() для селектора
  • # - выбирается тег с указанным далее id
  • . - выбирается тег, содержащий указанный далее класс
  • иначе - выбирается тег с таким наименованием

Во всех случаях берётся только первый найденный (по селектору) тег. Если же не находится - записывается сообщение об ошибке.

Кроме содержимого тега также копируется указанные в нём классы (а вот его атрибуты,- в т.ч. style,- игнорируются). После вставки атрибут o5include заменяется (для истории) на атрибут _o5include.

Теги <script> из вставляемых фрагментов НЕ обрабатываются (в этой версии)

Вставка может выполняться рекурсивно, т.е. скрипт обрабатывает ссылки во вставляемых фрагментах.

Посмотреть реализацию подгрузки разных фрагментов из двух разных файлов можно здесь, где для атрибута o5include показаны следующие вариатны значений:

  • './inc1.html?.cls2' — класс cls2, независимо от наличия квалификаторов;
  • './inc1.html?.cls2:aa' — класс cls2 с квалификатором aa (т.е. значение атрибута class должно содержать указанную подстроку);
  • './inc1.html?#id1' — с id='id1';
  • './inc1.html?span' — тег span;
  • './inc1.html?[moe]' — атрибут moe,- в квадратных скобках м.б. любое допустимое выражение;
  • './inc1.html?' — включение файла при пустом селекторе;
  • './inc2.html' — включение файла при отсуцтствии селектора

    • Вставляется содержимое первого тега (загруженного файла), удовлетворяющего заданному выражению. В примере были показаны ставки в теги div, однако можно вставлять в любые теги (дополняя при этом их содержимое).

    Разумеется, сами файлы inc1.html и inc2.html считываются только по одному разу. Их содержимое можно увидеть в протоколе в раскрывающемся сообщении 'Содержимое прочитанного ...'

    Ядро библиотеки проверяет наличии данного скрипта и тегов с атрибутои o5include и, при их наличии,- модуль o5inc выполняется в первую очередь, независимо от гположения его скрипта. ами При подключении скрипта совместно с иными библиотечными скриптами он должен располагаться прежде них. Также модуль может выполняться автономно,- без подключения ядра библиотеки o5com!.js,- как это сделано в упомянутом примере. Результаты загрузки файлов приводятся в сообщении "Обработка 'CInclude'" в console.log.

    После загрузки файлов, указанных в атрибуте o5include скриптом генерируется window сообщение olga5-incls, которое должно использоваться вмето (вместе) сообщения DOMContentLoaded.

    Скрипту м.б. задан атрибут o5getall. Его умалчиваемое значение 'true'. Если задать 'false', то вставка (и загрузка файлов) будет только в те теги, у которых самих и их предков НЕ задан стиль display = 'none'. Это может снизить трафик, однако при изменении их видимости следует генерировать событие window.dispatchEvent(new CustomEvent('o5include')) для (вероятной) вставки новых фрагментов.
