Организация зависаний элементов HTML-страницы

1. Общие сведения

Скрипт o5shp.js входит в библиотеку Olga5 и предоставляет авторам HTML-страниц возможность создания эффектов зависания блочных элементов, расширяя скудные возможности стандартного свойства position: sticky:

  • зависание блока внутри вложенных контейнеров;
  • групповое зависание блоков друг под другом, согласно приоритетам;
  • зависание блоков как вверху, так и внизу контейнера

Подобно остальным скриптам библиотеки данный скрипт может быть подключен не только к отдельным HTML-страницам, но и функционировать внутри Blogger'а, в частности на шаблоне Dynamic View. Скрипт выполняет обработку тегов с заданным классом olga5_shp, находящихся в областях действия его инициализации.

Скрипт пока не поддерживает CSS-атрибуты zoom и transform.

При скроллинге же самогó зелёного контейнера (полосками в его левой стороне) первый блок, содержащий полный текст песни (у него есть слева своя полоска скроллинга) и Youtube'вский видеоклип, зависнет вверху этого контейнера.

  • 'наезжанием' - куплет 5÷8 с квалификатором 'Over';
  • 'сталкиванием' - куплет 9÷12 с квалификатором 'Push';
  • 'стискиванием' - куплет 13÷16 с квалификатором 'Clench';
  • 'сдвиганием' - куплет 17÷20 (и последующие) с квалификатором 'Shift' либо без такового.

Временное отключение зависания блока (ради освобождения места), выполняется двойным кликом по его неактивной части. Например, для первого блока дваждыкликать следует где-то по тексту песни, т.к. клики по изображению перехватываются обработчиком Youtube'а.

3. Подключение скрипта

Подключение скрипта o5shp.js выполняется в заголовке (в секции <head>) HTML (или PHP) файла:

<script src='olga5/o5shp.js'></script>
...
<script src='olga5/o5com!.js' o5debug=2 o5init_events></script>

Здесь дополнительное (выделенное курсивом) подключение o5com!.js - обязательно.

Cкрипт создает событие 'o5shp_scroll',- только для тестов (внешним модулем './add/alltst.js')

4. Задание зависания блока

Для задания возможности зависания, например тега <div> можно написать:

<div id='div1' class="olga5_shp:1:Up" olga5_frames="id:shp-demo">

где в перечне классов атрибута class указан обязательный класс olga5_shp, после которого определены два (необязательных,- см. п. 4.1) квалификатора : Up и 1. Также, для тега задан атрибут (необязательный, дополнительный,- см. п. 4.2) olga5_frames.

4.1. Квалификаторы класса olga5_shp

Наименование класса может сопровождаться несколькими квалификаторами, уточняющими характер зависания. В приведенном примере для div'а квалификаторами 1 и Up указаны: уровень ='1', а его зависание - вверху контейнера. Квалификаторы разделяются символом ':', регистр букв игнорируется, пробелы не допускаются.

Предусмотрены следующие квалификаторы:

  • Top     — блок зависает вверху контейнера;
  • Bottom   — -"- внизу контейнера;
  • Up     — зависание происходит по достижению верха контейнера;
  • Down   — зависание происходит по появлению внизу контейнера;
  • Over  — наезжает-на предыдущий зависший блок (см. блок со стр. 5÷8);
  • Push   — сталкивает -"- (стр. 9÷12);
  • Clench   — стискивает -"- (стр. 13÷16);
  • Shift   — сдвигает -"- (стр. 17÷20 и дальше) - умалчиваемое значение;
  • i        — целое число,- уровень блока, определяющий возможность сдвига вышерасположенных блоков (при приближении к ним);
  • Alive  — автоматически восстанавливает зависание этого блока после прекращения возможности его сдвига последующим блоком;
  • Kill   — отключает (после своего появления на экране) зависание у находящихся выше блоков с таким же или большим уровнем;
  • Remove   — сдвигает предыдущий объект аналогично Shift, но при этом сам обычно не является подвисабельным;
  • None - отключение событий тега (см. также класс olga5_shp_none),- обработка не выполняется,- только визуальное отображение.

У всех квалификаторов значимыми (проверяемыми программой) являются лишь первые символы их наименований без учёта регистра.Эти символы здесь выделены (для наглядности) прописными жирными буквами.

По умолчанию обычно предполагаются умолчания (после olga5_shp) :Top:Up:Shift:0. Однако, при наличии квалификаторов Kill или Remove - предполагается отсутствие зависания и для его осуществления следует конкретно указывать направление: Up либо Down. Если заданы взаимоисключающие квалификаторы (T и B, или U и D, или O и P и S), то значащим будет крайний справа.

Класс olga5_shp c калификатором Remove может присваиваться любому тегу, в т.ч. даже блоку нулевых размеров. В вышеприведенном примере он использован в абзаце с текстом "Слова к последним стр. 25 ÷ 28:" (в конце примера) и сдвигает (этот последний) зависший куплет. При его прикосновении к зависшему тегу с неменшим значением уровня последний сдвигается, аналогично, как под действием квалификаторов Shift и Up. Квалификатор Remove игнорируется при наличии Kill.

Квалификаторы Push, Shift, Over, Alive и Kill срабатывают только если уровень у этого блока не больше, чем у предыдущего (т.е. расположенного выше по тексту) зависшего блока. В примере у блока со стр. 17÷20 уровень задан '2' и поэтому он не сдвигает предыдущий (у которого уровень= '1'), а прилипает к нему. И оба они (как и остальные куплеты) подвисают под блоком с YouTube-роликом, у которого уровень= '0'.

Вместо использования квалификатора Kill можно указывать для подвисабельного тега наименование владельца,- например olga5_owners="id:kst (см. п. 4.2), как это сделано в первом примере в п.2, где подвисший текст плавно удаляется с экрана.

4.2. Дополнительные атрибуты тега

Кроме задания класса olga5_shp, для подвисабельного объекта могут указываться дополнительные (специфические для данной библиотеки) атрибуты, каждый из которых определяет список из одного или нескольких контейнеров, ограничивающих его подвисание:

  • olga5_frames — контейнеры (например теги <div>), на ближайшей верхней или нижней границах которых будет подвисать объект при скроллировании документа или его части;
  • olga5_owners — контейнеры, ограничивающие видимость объекта дальнейшей из границ фреймов;
а также атрибут без значения:
  • olga5_repeat — дублирующий значения двух выше-названных атрибутов (если какой из них не задан) от ближайшего из предыдущих объектов того же уровня вложенности с непустыми значениями.

Действие перечисленных атрибутов можно посмотреть на приведенных ниже примерах (вызываемых в отдельном окне кликом овальных кнопок с надписью 'Показать').

Каждый из контейнеров, перечисленных в списках, определяется набором от одного до трёх полей: код (обязательное поле), значение и необязательный счётчик контейнеров. Поля разделяются символом ':'. Предусмотрены коды:

  • Id      — произвольный тег с id равным значению - любой блочный тег,- возможно даже (как исключение) выше верхнего уровня;
  • Class — ближайший по счётчику контейнер, содержащий классы, перечисленные (разделённые '.' или ',' без пробелов) в значениии;
  • Node  — аналогично,- контейнер с nodeName равным значению, где пустое значение подразумевает контейнер верхнего уровня;
  • Scroll — контейнером является ближайший (по выложенности) контейнер с видимой полосой скроллинга;
  • Bord — аналогично,- блочный контейнер с видимой границей,- т.е. имеющий рамку или или фон с отличимым цветом.

Здесь значимым также является лишь первый символ. Поле значение может отсутствовать, по умолчанию оно интерпретируется пустой строкой или нулём.

Контейнером верхнего уровня является окно браузера, либо ближайший обрамляющий контейнер, тег которого содержит класс olga5_shp. То есть, контейнеры ищутся среди 'предков', начиная от родительского тега до (включительно) либо тега body, либо контейнера, который сам является подвисабельным объектом.

Код I определяет произвольный tag с указанным id, который ищется во всём документе и при этом значение счетчика для него игнорируется. Тег может быть произвольным элементом и не обязательно являться контейнером для объекта. Предполагается, что такой тег должен иметь определяемые границы и фиксированную позицию на экране.

Использование кодов 'B' и 'S' продемонстрировано ниже,- в примере п.5.2. При этом 'S' может не давать однозначного результата т.к. при возможном изменении размеров контейнеров могут пропадать их полосы скроллинга (как можно увидеть ниже в примере п.5.2).

Значение счётчика по-умолчанию равно '1', при этом ищется 1-й (ближайший) контейнер, удовлетворяющий условию "код:значение". При значении '0' поиск контейнера не выполняется. Для подключения нескольких контейнеров с одинаковыми парами 'код:значение' нужно указать соответствующее количество контейнеров с различными значениями счетчиков. Если числовое значение записано с литерой 'F' в конце, то такой контейнер считается фиксированным

Если для olga5_owners не определён код, то принимается по-умолчанию значение B,- как если бы был указан ближайший контейнер с видимой границей. Граница блока считается видимой, если блок является <div> тегом и цвет его фона отличается от окружающего цвета (фона или границы - border'а).

Если же код не определён для olga5_frames, то умалчиваемым значением принимается S,- как если бы был указан ближайший скроллируемый контейнер, либо окно браузера,- тег BODY.

Если в документе не найдены контейнеры (теги) для заданных значений атрибутов, то выдаётся 'жёлтое' сообщение об ошибке "Не определены ссылки на фреймы/контейнеры подвисания" с перечнем ненайденных значений.

4.3. Реализация зависания

Зависание реализовано помещением зависаемого объекта в блочный контейнер div, который имеет фиксированное позиционирование. Место исходного объекта занимает его клон, которому добавляются: стиль opacity=0,- делающий его невидимым на экране; и класс olga5_IGNORE,- делающий его невидимым для функций GetTagsByClassNames() и SelectByClassName().

При скроллировании окна и изменении его размеров пересчитываются положение и размеры клона и определяют положения как контейнера на экране, так и объекта в контейнере.

Предусмотрена вложенность подвисабельных объектов, когда внутренние такие не выходят за предела такого внешнего (а самый внешний - окно браузера).

4.4. Отключение скрипта

Для отключений зависания блока один из содержащих его контейнеров должен иметь класс olga5_shp_none. Его действие аналогично использованию квалификатора :None.

Использование такого отключения м.б. целесообразным при включении html-фрагмента, содержащего подвисабельный тег (теги) с классом olga5_shp, в различные документы, в некоторых из которых подвисание не требуется.

5.4. К разработке новых модулей

Нижеследующее м.б. полезно при разработке новых модулей для библиотеки.

1. При использовании скрипта могут возникать ситуации, когда после каких-то 'сложных' манипуляций в других, не библиотечных, js-программах подвисавшийся тег будет как бы исчезать или наоборот,- оставаться на экране, хотя должен бы был 'уйти'.

Сие может вызываться тем, что полное позиционирование подвисабельных тегов выполняется лишь при изменении размера окна браузера. А вот при скроллинге пересчитывается только относительный сдвиг объектов.

Для того чтобы в создаваемых js-программах правильно обрабатывать такие ситуации следует имитировать изменение размеров экрана генерированием события:
window.dispatchEvent(new window.Event('resize')), прописанным в этих 'иных js-программах'.

Если в подвисабельных элементах задаются/изменяются свойста (связянные с размерами) DOM-элементов, которые определены по id, то такое же изенени должно быть продублировано для элементов с [id]_shdw!

Иначе размеры подвисабельного элемента и его прототипа могт оказаться различными, что исказит требуемое его размещение.
Вообще-то лучше свойства объектов внутри подвисабельных элементов определять через их классы, которые не требуется дублировать.

2. При использовании скрипта предусмотрено, что двойной клик на подвисшем теге временно освобождает его от прилипания к заданной границе. Поэтому, при написании кликовских обработчиков для объектов вложенных в подвисабельный тег следует в них (в обработчиках) предусмотреть:
event.cancelBubble = true;.

3. Чтобы подвисший объект не занимал весь экран или большую его часть следует объект поместить в контейнер, ссылку на который задать в атрибуте olga5_owners объекта, а размеры контейнера задав как для доступной части экрана.
