Пояснения к интерфейсу примера

Подвисание может выполняться на верхней или нижней границе одного или нескольких скроллируемых контейнеров div1, div2, div3 и/или div4.

Объект может фиксироваться только после того, как он появится на экране полностью. Объект не может быть одновременно фиксированным своими противоположными сторонами (при коллизиях приоритет у 'top' и 'left').

Незафиксированный объект c position:relative (таких тут большинство и курсор на них принимает форму 'лапы') можно перетащить по экрану, "захватив" его курсором мыши.

Настройка и активация выполняется CheckBox'ами, расположенными на правом краю экрана.

Три верхних (clon, cart и outline) действуют сразу на все подвисабельные объекты. Для остальных объектов CheckBox'ы собраны в группы.

Для каждого подвисабельного тега (объекта) указываются: его номер, level, alive, 4 места подвисания, один (из 4-х) способов сдвига и список контейнеров, с которыми он взаимодействует.

Назначение каждого поля показывается во всплывающей подсказке после наведения курсора мыши.

Для каждого контейнер из списка (body, div1 и т.д.) указываются пара CheckBox'ов: 'f' и 'c', определяющик его как подвисающий и/или обрезающий.

Все CheckBox'ы обычно активны только если объект является (частично) видимым, а 'clon' и 'cart' отрабатыват только после того, как объект был зафиксирован.

Подвисабельный объект имеет желтый фон с красной рамкой, создаваемой атрибутом "border: 12px solid red;". Синяя рамка создаётся атрибутом "outline: solid 8px blue;",- она находится как бы вне объекта и поэтому остаётся видимой, пока видеи хоть один пиксел тега.
Изменение опции outline выполняется корректно только для объектов, которые еще не были зафиксированы.

На самих объектах отображаются параметры (заданные CheckBox'ами) :

  • в первой строке - имя тега;
  • во 2-й - условия его подвисания, для наглядности разделённые символом ':'
  • в 3-й - и 4-й перечислены фреймы на которых выполняется подвисаюние и обрезание

Взаимное положение тегов в премере можно менять просто перетаскивая (не зафиксированные) объекты по экрану, "захватив" его курсором мыши.
Следует избегать наложения объектов, т.к. при этом может некорректно изменяться их перекрытие (определяемое zIndex'ом).

При натыкании объектов друг на друга могут появляться их "тени"Ю показывающие положение одного из прототипаов. Этот эффет появляется при открытии страницы с ключем o5debug

Для упрощения логики демонстрации подвисаний отключена возможность скроллироания контейнеров "перетягиванием" их курсором мыши. Т.е. скроллинг выполняется либо полосками скроллинга, либо колёсиком мыши (с нажатием Shift - по горизонтали).
Для имитации (при отладке) скроллирования контейнеров в правом верхнем углу показаны "желтые" кнопки и выбранный контейнер.
