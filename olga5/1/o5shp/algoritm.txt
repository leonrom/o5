        - сброс hidden


ФИКСАЦИЯ-НА-ГРАНИЦЕ
попытка фиксации тегов поочередно на каждой внутренних границ заявленных контейнеров

- ЦИКЛ по очереди, согласно направленности проверяем aO5 [при всех проверяем ЕСЛИ только для НЕ hidden тегов]:
    - ЕСЛИ тег достиг ближайшей границы
        - ЕСЛИ тег достиг какую-то из своих границ
            - ФИКСАЦИЯ aO5
            - вызов проверки на ВЗАИМОДЕЙСТВИЕ (aO5)
    - ИНАЧЕ
        - break


ВЗАИМОДЕЙСТВИЕ (aO5)
выполняется для только что зафиксированного тега aO5

- ЦИКЛ по fO5  начиная со следующего за aO5 проверяем на поджимание
    - ЕСЛИ fO5 достиг aO5
        -ЕСЛИ у fO5 приоритет больше (чем aO5) и ЕСЛИ имеется ЗАТЕНЕНИЕ:
            - пересчитываем размеры aO5
                - ЕСЛИ  размер<0 и если НЕ alive 
                    - для  aO5 включаем  hidden                 
                - ИНАЧЕ 
                    - отлючаю hidden                 
            - в fO5.sticks сохраняю поджатие aO5
    - break     // тут можно сразу, т.к. требуется менять лишь aO5, а больше всего подожмёт именно первый-ближайший

- ЦИКЛ по fO5 начиная со следующего за aO5 проверяем с меншим приоритетом на прилипание
    - ЕСЛИ fO5 достиг aO5
        - ЕСЛИ у fO5 приоритет меньше и ЕСЛИ имеется ЗАТЕНЕНИЕ:
            - пересчет положения fO5
            - ФИКСАЦИЯ fO5
            - вызов проверки на ВЗАИМОДЕЙСТВИЕ для всех в fO5.sticks
            - вызов проверки на ВЗАИМОДЕЙСТВИЕ (fO5)
    - ИНАЧЕ - break     // нельзя break только по 'достиг', т.к. м.б. есколько достигших одновременно


при сранении размеров: для зафиксированных брать posC, а для незафиксированных - posO
процедуры DoFix и UnFix выполнять в 2-х режимах: 
    отметка о фиксации (во время проверок)
    и реальная фиксация/расфиксация   (после всех проверок)

ФИКСАЦИЯ = установка отметки о фиксации без фактической фиксации
СРАВНЕНИЕ=
    - ЕСЛИ ОтмеченДляФиксации то
        - rez = ЕСЛИ приоритет aO5 меньше чем fO5 ТО -1 иначе
                ЕСЛИ приоритет aO5 больше чем fO5 ТО +1
        - если rez!==0 и есть КАСАНИЕ и ЗАТЕНЕНИЕ 
          и нету "ближайший" фрейм или aO5 ближе "ближайшего" фрейма 
            - return rez
    - return 0

            - ЕСЛИ тег достиг границы фрейма
                - проверяем (и сохраняем) как "ближайший" фрейм

        - для fO5 тех, что после aO5 в обратной последоветельности 
            - ЕСЛИ СРАВНЕНИЕ < 0 тогда 
                - ФИКСАЦИЯ aO5 (как прижатие к fO5)
                - запоминаю cO5 = fO5
                - break 

        - для fO5 тех, что после aO5  в обратной последоветельности  (т.е. такой же цикл)
            - если fO5===cO5 
                - break
            - ЕСЛИ СРАВНЕНИЕ > 0  тогда                 
                - меняю размер fO5
                    - ЕСЛИ для fO5 размер<0 и если НЕ alive то ему
                        - включаем  hidden                 
                    - ИНАЧЕ

    - для aO5 всех в любой последоветельности

- для aO5 всех в любой последоветельности
